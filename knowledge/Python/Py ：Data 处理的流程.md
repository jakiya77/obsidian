### 第一层：I/O 传输层（从硬盘到内存的桥梁）

- **代表人物**：`open()`, `fp`
    
- **干什么**：负责和操作系统打交道，把硬盘上的 0 和 1 变成 Python 能认识的“原始字符串”。
    
- **特点**：数据是连成一坨的，带着各种格式符（如 `\n`, `\t`）。
    ![[png：20260409111921.png|425]]
    
    

### 第二层：解析层（协议翻译官）

- **代表人物**：`csv.reader()`, `json.loads()`, 图片解码器
    
- **干什么**：给原始字符串定规矩。如果是 CSV，它就找逗号；如果是 JSON，它就找大括号；如果是图片，它就翻译像素点。
    
- **特点**：通常使用“懒加载”（水龙头模式），极度节省内存。
    ![[png： 20260409112036.png|413]]
### 第三层：容器层（Python 的原生大仓库）

- **代表人物**：`list()` (列表), `dict()` (字典)
    
- **干什么**：把解析好的零碎数据，一次性全部打包放进内存里，方便你用 `for` 循环遍历，或者用索引 `[]` 抽查。
    
- **特点**：什么都能装，非常灵活，但是**不能做数学运算**，因为里面装的可能还是文本。
    ![[png：20260409112203.png|411]]

### 第四层：计算层（专为 CPU/GPU 打造的数学引擎）

- **代表人物**：`np.array` (NumPy 矩阵), `torch.Tensor` (PyTorch 张量), `pd.DataFrame` (Pandas 数据框) 
       
- **干什么**：把装在第三层里的数据，剥离掉杂质（比如转成纯 float），排兵布阵变成底层的 C/C++ 内存块，开启疯狂运算。
    
 - **特点**：速度极快，支持矩阵切片、矩阵乘法，但要求数据必须规整（比如必须全是数字，长宽必须对齐）。

🔥 **【进阶笔记：第四层内部的生态壁垒与转换】

虽然 `np.array` 和 `torch.Tensor` 在数据结构上是平级的，但在深度学习工程中，它们有极其严格的内外场之分。数据必须经过一次交接仪式：

- 场外预处理区（CPU 专属）：np.array
    
    - **定位**：科学计算的原住民，负责把原始数据切好。
        
    - **数据格式**：默认分配 **64位双精度浮点数 (`float64`)**。
        
    - **局限**：只懂 CPU 计算，不懂神经网络和反向传播，进不了显卡。
        
- **场内炼丹炉（GPU 专属）：`torch.Tensor`**
    
    - **定位**：PyTorch 的“专属货币”。神经网络极度挑食，只吃 Tensor。
        
    - **数据格式**：通常被强制转换为 **32位单精度浮点数 (`float32`)**（代码：`torch.FloatTensor()`）。
        
    - **为什么从 64 位降级到 32 位？**：牺牲掉小数点后十几位（神经网络根本不需要）的无用精度，直接换来**显存占用省一半**、**矩阵运算速度翻倍**！
        
    - **独家特权**：只有 Tensor 才能拿到深度学习的两张顶级门票：
          **进驻显卡 (`.to(device)`)** 
		  **自动求导追踪 (`Autograd`)**。

**💡 黄金流水线公式：**

1. 用 `np.array(...).astype(float)` 把数据从文件里挖出来。
    
2. 用 `torch.FloatTensor(...)` 进行包装转换（。
    
3. 送进 `NeuralNet` 炼丹（**上流水线**）。

>[!hint]- open(path,'r')
>- **`'r'` (Read / 只读)**：默认模式。只能看，不能改。如果文件不存在，程序会报错。
>- **`'w'` (Write / 只写)**：**危险操作！** 它是用来写新文件的。如果这个文件已经存在，Python 会**毫不犹豫地清空里面的所有内容**，然后从零开始写。
>- **`'a'` (Append / 追加)**：也是写入，但它比较温和。它不会清空原文件，而是把新内容顺着追加到文件的最末尾（常用于写日志文件）。
