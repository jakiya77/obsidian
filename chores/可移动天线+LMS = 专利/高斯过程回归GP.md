# 1) 一句话概念

**高斯过程回归（Gaussian Process Regression, GPR）** 是对“未知函数”做**概率建模**的方法：把函数 (f(x)) 看成“来自某个多维高斯分布的随机函数”。给定少量带噪观测，它能给出任意新点的**预测均值**（函数值估计）和**预测方差**（不确定度）。

# 2) 形式化与直觉

- 观测模型：
$$
    (y_i = f(x_i) + \varepsilon_i ，\varepsilon_i\sim\mathcal N(0,\sigma_n^2))
$$
    
- 先验：
$$
f(\cdot)\sim \mathcal{GP}\big(m(\cdot),k(\cdot,\cdot)\big)，常取 (m(\cdot)=0)
$$
    
$$
用 X=[x_1,\dots,x_n])、(y=[y_1,\dots,y_n]^\top，核矩阵 K_{ij}=k(x_i,x_j)
$$
    
- 对新点 x_* 的后验预测：  
    ![[Pasted image 20251020155233.png]]
    

常见核：

- RBF/SE：
$$
(k=\sigma_f^2\exp(-|x-x'|^2/2\ell^2))（很平滑）
$$
    
- Matérn(5/2)：较“粗糙”，对真实黑盒更稳
    
- ARD：为每个维度配一个 (\ell_d) 做“自动相关性选择”
    

**超参数学习**：最大化对数边际似然（MLL），相当于“用数据自适应选核参数与噪声强度”。

# 3) GPR vs. 贝叶斯优化（BO）：关系与区别

| 维度   | GPR                      | 贝叶斯优化（BO）                                  |
| ---- | ------------------------ | ------------------------------------------ |
| 目的   | **拟合**未知函数（回归 + 不确定度）    | **最小/最大化**昂贵黑盒函数                           |
| 关系   | —                        | **把 GPR 当作 surrogate（代理模型）**               |
| 产物   | μ(x),σ(x)均值+方差）          | **采集函数**（EI/PI/UCB 等），用 (\mu,s) 决定“下一点去哪测” |
| 使用场景 | 需要对函数整体形状与置信区间建模         | 每次评估代价高（仿真/实验），样本预算小                       |
| 迭代   | 一次性拟合或增量更新               | **循环**：拟合GP → 最大化采集函数 → 评估黑盒 → 更新GP        |
| 优缺点  | 表达不确定度、样本效率高；训练 (O(n^3)) | 能“自动平衡探索/开发”，适合低维<=20 黑盒全局优化               |

**结论**：GPR 是建模工具；BO 是优化框架 —— **BO 常以 GPR 为内核**。

# 4) 采集函数（BO 的“决策器”）
以**最小化**为例，设当前最优 (f_\text{best})，在 (x) 处 GP 预测 (\mu(x), s(x))：

- **EI（期望改进）**：
$$
\mathrm{EI}(x)=\mathbb E[\max(0, f_\text{best}-f(x))] 
    
    
$$
$$
闭式解依赖标准化 (z=\frac{f_\text{best}-\mu}{s})（兼顾“好均值”和“大方差”）
$$

- **UCB**：
$$
 \mathrm{UCB}(x)=\mu(x)-\kappa s(x)（(\kappa) 大→更爱探索）
$$
    
- **PI**：
$$
只看超过 f_\text{best} 的概率（贪心些）
$$
    

# 5) 具体例子 A：1D 玩具函数（直观）

目标：
(\min_x f(x)=\sin(1.5x)+0.1\mathcal N(0,1)), (x\in[0,6])  
流程：

1. 先取 3–5 个初始点评估 (y)，拟合 GP（如 Matérn 5/2）。
    
2. 计算 EI，在 ([0,6]) 上找 (\arg\max \mathrm{EI}(x)) 得下一点 (x_1)。
    
3. 评估 (f(x_1))，把 ((x_1,y_1)) 加入数据，更新 GP。
    
4. 重复 20–40 次；最终在“少评估”的前提下逼近全局最优。  
    体会：**有不确定度的地方、且均值看起来也不错**，EI 会主动去试 —— 这就是“探索/开发”的自动平衡。
    

# 6) 具体例子 B：调中间阵元位置 (x) 最大化 SINR

- 黑盒：(g(x)=\text{SINR}(x))（含 **LMS 收敛抖动、信道随机性** → 有噪声）
    
- 变量：(x\in[x_{\min},x_{\max}])（1 维，完美适配 GP-BO）
    
- 做法：
    
    1. 以 **GPR** 建模 (g(x))，用 (y=-g(x)) 转成“最小化”。核建议 **Matérn(5/2)+噪声项**；把 (x) 线性归一化到 ([0,1])。
        
    2. 初始评估 5–8 个 (x)（均匀或拉丁超立方）。
        
    3. 用 **EI 或 EI+** 做采集；每轮挑 1 个新 (x) 评估（可把每次仿真平均 (R) 次以降噪）。
        
    4. 迭代 ~30–50 次，取历史上 **最好的 (x)** 作为答案。
        
- 经验小贴士：
    
    - **观测噪声不小**时，把同一 (x) 复测 2–3 次并求均值，或采用“异方差 GP”。
        
    - 出现多峰/非平稳时，Matérn 核比 RBF 更稳；**EI+**（在 EI 上加小“乐观偏置”）能减少被“过早乐观”困住。
        
    - 核矩阵数值不稳定时加 **jitter**（如 (10^{-6})）到对角线。
        

# 7) 何时用 / 何时别用

- **很合适**：评估开销大（仿真分钟级/小时级）、维度低（(\le 10) 更佳）、响应平滑、噪声中等。
    
- **不太合适**：高维（(\ge 30)）、强离散/组合、剧烈非平稳。可考虑：分解/加法 GP、随机嵌入、Trust-Region BO（TuRBO）、或改用 TPE/随机森林/进化算法。
    

# 8) 常见坑（快速排雷）

- **过小长度尺度** → 过拟合、方差虚高；多跑几轮 MLL 重新初始化超参。
    
- **未标准化** → 数值不稳；请把 (x) 映射到 ([0,1])，(y) 做零均值/单位方差。
    
- **噪声忽略** → 方差估计失真；务必建 (\sigma_n^2)。
    
- **样本太多**（(n\gg 2000)）→ (O(n^3)) 训练吃不消；用稀疏/诱导点 GP 或换 surrogate。
    

# 9) 迷你“上手清单”

1. 选核：Matérn(5/2)+ARD（多维时）+噪声项
    
2. 预处理：(x) 归一化，(y) 标准化
    
3. 初始化：5–10 个点
    
4. 采集函数：EI 或 UCB（(\kappa) 从 1.5–3 调）
    
5. 每轮：最大化采集函数 → 评估 → 更新 GP
    
6. 终止：评估预算用尽或改进<阈值；**返回历史最优点**而不是最后一轮的点
